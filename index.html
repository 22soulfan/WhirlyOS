NT Rebirth

MyriadOS

ReactOS // Light-Aesthetic
Target: $1\text{GB}$ - $2\text{GB}$ RAM

Optimized for older hardware. MyriadOS provides a reduced aesthetic layer that keeps the UI modern and responsive without the heavy resource cost of typical Windows clones.

Rolling Release

RenardOS

openSUSE // Light-Aesthetic
Target: $1\text{GB}$ - $2\text{GB}$ RAM

High-speed rolling release based on Tumbleweed. Uses a stripped-back aesthetic layer to ensure the kernel's speed isn't choked by the desktop environment.

Managed OS

RPandaOS

Cosmos C# // Light-Aesthetic
Target: $1\text{GB}$ - $2\text{GB}$ RAM

Experimental C# environment. The aesthetic layer is kept simple and optimized for managed memory, ensuring $1\text{GB}$ RAM stability.

Educational

WhirlyOS

Debian // Zero-Aesthetic
Target: Ultra-Minimal

Strictly functional. WhirlyOS removes the aesthetic layer entirely to maximize educational tool performance on legacy hardware. No bloat, no distractions.

RAM & Resource Roadmap

Our development priority is **Hardware Reclamation**. By utilizing differing levels of "Aesthetic Layers," we target specific RAM profiles for maximum efficiency.

ZERO AESTHETIC (Functional)

  • WhirlyOS (Debian Core)
  • Designed for sub-512MB RAM tasks

LIGHT AESTHETIC (Responsive)

  • MyriadOS / RenardOS / RPandaOS
  • Targeting $1\text{GB}$–$2\text{GB}$ RAM stability
